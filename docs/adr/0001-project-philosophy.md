# ADR-0001: Minecraft AI Studio Philosophy

Status: Accepted

Date: YYYY-MM-DD

---

## Context

Minecraft AI Studio aims to help players design Minecraft worlds through collaboration between specialized AI roles.

Early development focused on creating many Markdown documents.

As the project evolved, it became clear that long-term value comes from reusable knowledge, structured workflows, and real examples rather than document quantity.

---

## Decision

The project will prioritize:

1. Reusable knowledge.
2. Clear workflows.
3. Real project examples.
4. Human creativity over AI automation.

AI should guide creativity, not replace builders.

---

## Consequences

Positive:

- Easier for contributors to understand project goals.
- Consistent future decisions.
- Better long-term maintainability.

Negative:

- Progress may appear slower.
- More effort spent on documentation quality.
- Requires regular review of existing material.

---

## Alternatives Considered

Continue producing additional prompts and knowledge documents without formal architecture decisions.

Rejected because it would eventually create inconsistency.
